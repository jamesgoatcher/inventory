JPG.Inventory is simultaneously a visual item management app and a vanity project born out of intense collecting and obsessive organization.
The Inventory app is powered by a rather large JSON file with two root nodes. One node contains the folder structure and labels needed to keep the app organized and the other contains the individual item details.
Updating a large JSON file manually is both tedious and dangerous. Making programmatic updates to the manifest greatly reduces the chance of error and the visual component makes the changes more relatable.
